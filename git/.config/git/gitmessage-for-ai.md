# コミットメッセージ生成に関する指示

以下のルールに従って、日本語のコミットメッセージを生成してください。

---

## 基本ルール

- **メッセージは日本語で書いてください。**
- **コミットメッセージの先頭に、変更の種類に応じた絵文字とプレフィックス（分類）を必ず付けてください。**
- **主語は「変更されたコンポーネントや機能」を意識してください。**
- **最初の1文で変更内容を要約してください（サマリー）。**
- **絵文字とプレフィックスは、サマリーの行にのみ付けてください。**
- **必要に応じて、サマリーの下に「変更内容の詳細」を箇条書きで記述してください。**
- **複数の変更がある場合でも、要点を簡潔にまとめてください。**

---

## 絵文字とプレフィックス一覧（変更種別）

| 絵文字 | プレフィックス | 内容 | 例 |
|--------|----------------|------|-----|
| 👍     | feat           | 機能の追加、新しい振る舞い | 👍 feat: ユーザー登録ページを追加 |
| 🐛     | fix            | バグ修正 | 🐛 fix: ログイン時に発生するバリデーションエラーを修正 |
| 📚     | docs           | ドキュメントの追加・修正 | 📚 docs: API仕様の記述をREADMEに追加 |
| ✨     | style          | スタイルやフォーマットの調整（動作に影響しない） | ✨ style: コード整形と不要な空行を削除 |
| 🔨     | refactor       | 動作に影響しないコード改善・リファクタリング | 🔨 refactor: ボタンコンポーネントを関数分割 |
| 🚀     | perf           | パフォーマンス改善 | 🚀 perf: リスト表示のレンダリング速度を向上 |
| 🧪     | test           | テストの追加・修正 | 🧪 test: バリデーションユーティリティのテストを追加 |
| 👷     | build          | ビルド構成・依存関係の調整 | 👷 build: webpackの設定を変更 |
| 🤖     | ci             | CI/CD構成の変更 | 🤖 ci: CircleCIのジョブを修正 |
| 🔧     | chore          | 雑務的な作業（設定・依存更新など） | 🔧 chore: TypeScriptのバージョンを更新 |
| ⏪     | revert         | 以前のコミットを元に戻す | ⏪ revert: 不具合のあったレイアウト変更を取り消し |

---

## 書き方の指針

- **変更は「コンポーネント単位」または「機能単位」で記述してください。**
    - ✅ OK: 「ユーザーカードコンポーネントのレイアウトを修正」
    - ❌ NG: 「細かい修正をいくつか」
- **変更内容が一目で分かるよう、簡潔で具体的な表現にしてください。**
- **曖昧な言葉（例：とりあえず、ちょっと）、主観的表現は避けてください。**

---

## 実際の運用例

### 単一の変更


```text
👍 feat: プロフィール編集機能を追加
```

### バグ修正とリファクタリング（詳細を追記）
```text
🐛🔨 fix/refactor: ユーザー一覧の表示不具合修正と描画ロジック整理

- APIレスポンスにnullが含まれるケースを考慮
- 冗長なmap関数をループに置き換え
```

### スタイル調整とドキュメント更新
```text
✨ style: コードフォーマットをprettierに準拠  
📚 docs: コーディング規約の更新
```

### テストの追加（詳細付き）
```
🧪 test: ログイン機能に対するテストケースを追加

- 正常系（成功時）のテストを追加
- 異常系（パスワード未入力）のテストを追加
```
